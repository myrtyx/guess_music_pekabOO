# pekabOO — Product Requirements Document

## Что это

Мультиплеерная музыкальная викторина. Ведущий настраивает доску с треками (YouTube/SoundCloud), игроки угадывают мелодии. Два режима: классический (команды по очереди) и баззер (каждый сам за себя, Kahoot-стиль).

---

## Режимы игры

### Классический

```
Админка → Создать игру → Назначить треки → Запустить (классика)
    → /setup: создать команды
    → /: доска — команда выбирает ячейку
    → /select: музыка играет, таймер, 3 кнопки (все баллы / половина / не угадали)
    → /reveal: название трека + исполнитель
    → обратно к доске, ход следующей команды
```

- Команды ходят по очереди
- "Рандом" — случайная ячейка, x1.5 очков (3 использования на команду)
- Таймеры: `play_duration` (музыка) + `guess_duration` (ответ)

### Баззер

```
Админка → Запустить (баззер) → /host: код комнаты
    Игроки: /play → вводят код + имя → лобби
    Ведущий: "Начать игру" → выбирает первую ячейку
```

**Поток раунда:**
```
1. Музыка играет, "Слушайте..." [15 сек или ведущий открывает вручную]
2. БАЗЗЕР ОТКРЫТ [guess_duration сек, замораживается при ответе]
3. Игрок нажал BUZZ → [20 сек на устный ответ]
4. Ведущий судит:
   - Все баллы → +очки, reveal, победитель выбирает следующую ячейку
   - Половина → +очки/2, то же
   - Не угадали → -штраф, баззер снова открыт для остальных
   - Пропустить → ячейка закрыта, без очков
5. Если все ответили неверно → ведущий выбирает
6. Если время вышло → ведущий выбирает
```

**Выбор ячейки:** Победитель прошлого раунда (15 сек таймаут → ведущий).

**Пауза:** Замораживает ВСЁ — таймеры, музыку, баззер. Оверлей "ПАУЗА" на всех экранах.

**Переподключение:** Игрок может вернуться по имени. Хост — по URL `/host/<код>`.

## Страницы

| URL | Назначение | Кто видит |
|-----|-----------|-----------|
| `/admin` | Управление играми, треками, настройками | Ведущий |
| `/` | Активная игра (доска классики или редирект на /host) | Ведущий (экран/проектор) |
| `/setup` | Создание команд (классика) | Ведущий |
| `/select/<cat>/<pts>` | Ячейка с музыкой и таймером (классика) | Ведущий |
| `/reveal` | Название трека после ответа (классика) | Ведущий |
| `/host` или `/host/<код>` | Экран ведущего баззера | Ведущий (экран/проектор) |
| `/play` | Телефон игрока баззера | Игроки |

## Телефон игрока (`/play`)

**Вход:**
```
      pekabOO
  Код: [______]
  Имя: [______]
      [Войти]
```

**Ожидание:**
```
      Слушайте...
      TikTok 300
```

**Баззер:**
```
         14с

     ┌─────────┐
     │         │
     │  BUZZ   │
     │         │
     └─────────┘
```

**Между раундами:** доска + лидерборд. Если этот игрок выбирает — ячейки кликабельны.

## Хранение данных

```
games/
├── party_2024.json           ← настройки (жанры, очки, треки, таймеры)
├── party_2024_session.json   ← прогресс (команды, доска, ход)
├── new_year.json
└── new_year_session.json
```

Баззер-комнаты хранятся только в памяти сервера (не персистятся).

## Музыка

- **YouTube** — IFrame API, видео скрыто, только звук
- **SoundCloud** — Widget API, iframe скрыт
- Автоопределение по URL (`soundcloud.com/` → SC, остальное → YT)
- Стартовая секунда настраивается для каждого трека
- Проверка доступности в админке через реальный embed-тест (не oEmbed)
- Громкость: localStorage, всегда видна, применяется при каждом новом треке

## Технологии

- **Backend:** Flask + Flask-SocketIO + eventlet
- **Frontend:** Vanilla JS, Jinja2 шаблоны
- **Стиль:** Тёмная тема, Outfit + JetBrains Mono, фиолетовый/зелёный/янтарный
- **Хранение:** JSON файлы в `games/`
- **Деплой:** (планируется) Домен куплен, DNS настраивается

## Таймеры и значения по умолчанию

| Параметр | Значение | Где |
|----------|----------|-----|
| Время проигрывания музыки | 30 сек | Классика |
| Время на ответ | 30 сек | Оба режима |
| Автооткрытие баззера | 15 сек | Баззер |
| Время на устный ответ | 20 сек | Баззер |
| Время на выбор ячейки | 15 сек (+4 сек задержка) | Баззер |
| Штраф за неверный ответ | 50% очков ячейки | Баззер |
| Макс. игроков | 20 | Баззер |
| Рандом множитель | x1.5 | Классика |
| Рандом использований | 3 на команду | Классика |
